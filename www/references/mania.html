JSP以外のビューを使う

Designerクラスを利用してmobyletの機能を活用する

Mobyletは特定のビューに依存していません。
ビューのロジックは各Designerクラスとして独立しており、
JSP用のカスタムタグも、内部でDesignerクラスのロジックを呼び出しています。

そのため、VelocityなどJSP以外のビューを使用する場合でも、各Designerクラスを使用することで、
Mobyletの便利なビュー側の機能を使用することが可能です。

Designerクラスは、mobylet-core.jar内の「org.mobylet.view.designer」パッケージ配下に用意されています。

Config Injection

プロパティファイルを配置してDesignerをカスタマイズする

Designerクラスを独自にカスタマイズする場合、環境依存の値などを
設定するために、プロパティファイルを使用したいケースが存在します。

Mobyletでは、そのようなケースを簡単に実装する方法を用意しています。

MobyletInjectionConfigクラスを継承したConfigクラスを作成する

例として、MobyletConfigというクラスをMobyletInjectionConfigクラスを継承して作成します。
デフォルトではMobyletConfigに対応するプロパティファイルは
クラスパス配下の「フルパッケージ名 + .MobyletConfig.properties」となりますが、
getConfigNameメソッドを拡張することで、任意の名称を使用することが可能です。

プロパティファイル名を「mobylet.properties」とする場合は、以下のように記述します。 

public class MobyletConfig extends MobyletInjectionConfig {
	@Override
	public String getConfigName() {
		return "mobylet.properties";
	}
}

プロパティファイルの値を読み込む

クラスパス配下に「mobylet.properties」というファイルを作成し、キーと値を記述します。 

base.dir=/mobylet

ロジック内でプロパティファイルの値を読み込むには、以下のように記述します。 

MobyletConfig config = new MobyletConfig();
String baseDir = config.getConfig().getProperty("base.dir");
